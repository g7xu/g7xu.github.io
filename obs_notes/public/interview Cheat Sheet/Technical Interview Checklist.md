A mental model, I personal has when doing a problem solving (classic coding) interview
# General Workflow
- **Beginning**: General intro and conversation
- **Problem understanding**: understand problem (5 mins)
- **Solution Planning**: plan out ur approach (10 mins)
- **Solution Implementation**: ur plan to actual code (15 mins)
- **Explanation**: explain and clarify your approach with interviewer (5 mins)
# Beginning
- [ ] conversation to **set the tone** and **understand interviewees' style**
- [ ] inform your problem solving style to interviewer (I plan and then move onto solving)
- [ ] (optional): learn more about the interviewee
# Problem understanding
- [ ] expected input and output of the problem
	- [ ] Data type
	- [ ] Data constrain
- [ ] the expected time and space complexity
- [ ] walk through the problem with a few test cases
# Problem planning
- [ ] using pseudo code and general terms (reference to [[Algorithm CheatSheet (TODO)|here]])
- [ ] discussing the corresponding time complexity and space complexity
- [ ] discussing the edge case
	- [ ] empty input
	- [ ] duplication handling
	- [ ] null root (graph problem)
	- [ ] termination condition (recursion)
# Solution Implementation
- [ ] use common variable (`res`, `l`, `r`)
- [ ] use implemented func (reference to [here](https://neetcode.io/courses/lessons/python-for-coding-interviews))
- [ ] use helper func if you can
# Explanation
- [ ] walk through your approach
- [ ] explain the time complexity and space complexity again
- [ ] clean up the code if possible
